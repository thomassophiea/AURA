Context-Aware Search Behavior

The UnifiedFilterBar needs a search input that adapts to the page. How should it work?

CURRENT BEHAVIOR BY PAGE
Access Points
Searches: serial, name, model, IP, site
Connected Clients
Searches: hostname, MAC, IP, AP, site
Configure Networks
Searches: network name + band/security/status dropdowns
Dashboard / Insights
No text search — only dropdowns (site, time, environment)

How should the unified search input adapt?

The filter bar will always show Site + Environment + Time Range. The question is about the search box.

A

Always visible, context-aware placeholder

Search input is always shown. Placeholder text changes per page:
AP page → "Search APs by name, serial, model..."
Client page → "Search clients by hostname, MAC, IP..."
Dashboard → "Search across network..."
Each page passes a searchFields config and a local filter function.

B

Optional — pages opt in

Search input only appears when the page provides a searchConfig prop. Dashboard/Insights pages show just the dropdowns. Table-heavy pages (AP, Clients) show the search box. Cleaner for pages that don't need text search.

C

Integrated into the ContextualInsights popover

No standalone search box in the bar. Instead, clicking the context selector opens the popover with tabs + search (like your current Insights pattern). The search lives inside the popover, searching within whichever tab is active.
