Should the unified bar include the context tabs?

A

Yes — the Site dropdown becomes a context selector popover

Replace the simple "Site" dropdown with the rich popover. When on the AP page, clicking it defaults to the AP tab. On Clients page, defaults to Client tab. On Dashboard, defaults to Site tab. The popover lets you drill into Site → AP → Client from anywhere.

Pros

  • Powerful cross-page context switching
  • Consistent drill-down everywhere

Cons

  • More complex — might be overkill on simple config pages
  • Heavier component
B

Optional — pages opt in via a prop

By default, the bar shows a simple Site dropdown (like current FilterBar). Pages that want the rich context tabs pass showContextSelector={true}. Insights and Dashboard get tabs; AP, Clients, and config pages get the simple dropdown.

Pros

  • Right-sized for each page
  • Config pages stay simple

Cons

  • Two modes to maintain
  • Slightly less unified
C

Drop the tabs — just keep the rich Site popover with search

The Site dropdown becomes a searchable popover (search + scrollable list + status indicators) but WITHOUT the AI/AP/Switch/Client tabs. Those tabs only made sense in the Insights context. The AP/Client search already lives in the main search input.

Pros

  • Simpler, no tab confusion
  • Search box handles AP/client filtering

Cons

  • Loses the drill-down navigation
